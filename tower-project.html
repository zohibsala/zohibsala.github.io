Tower

Overview

The Tower project is a procedural destruction and visual effects project developed using Houdini, focused on creating a destructible historical structure for real time game engines.

The goal was to simulate a believable tower collapse using a combination of rigid body dynamics, procedural fracturing, debris generation and atmospheric dust effects. The project was designed as a contained scene to demonstrate how multiple VFX techniques can be combined into a single cohesive system suitable for game development.

The visual direction was inspired by historical architecture seen in games such as Assassin's Creed, with an emphasis on realism, weight and physical behaviour rather than exaggerated destruction.

My Role

I was responsible for the full development of the project, including modelling, procedural setup, simulation design and engine integration planning.

Key Features

  • Procedural destruction system built using Houdini
  • Rigid body dynamics simulation for realistic structural collapse
  • Voronoi and material based fracturing for controlled breakage
  • Constraint based system to simulate structural resistance and staged collapse
  • Secondary debris generation using particle systems
  • Dust and smoke simulation using volumetric effects
  • Houdini Digital Asset with exposed parameters for real time control
  • Designed for integration into Unity and Unreal Engine pipelines

Technical Implementation

The base tower model was created in Autodesk Maya and then imported into Houdini for procedural destruction and simulation work. Houdini was used due to its node based workflow and strong support for procedural VFX pipelines.

The destruction system was built as a layered process. The tower was fractured using Voronoi techniques to create both large structural pieces and smaller fragments. Constraint networks were applied to control how different parts of the tower held together and broke apart, allowing for a more realistic staged collapse rather than immediate destruction.

Rigid body dynamics were handled using the RBD Solver, which simulated gravity, collisions and debris interaction. Packed geometry was used to optimise performance while maintaining a high number of fragments.

Secondary debris was generated using particle systems, allowing smaller fragments to be emitted during the collapse. These were optimised using instancing techniques to reduce performance cost.

Dust and smoke effects were created using volumetric workflows and the Pyro Solver. These effects were driven by the destruction simulation to ensure they reacted naturally to the collapsing structure.

The entire system was designed procedurally and packaged as a Houdini Digital Asset, allowing key parameters such as scale to be adjusted externally for reuse in a game engine.

Challenges

One of the main challenges was learning Houdini for the first time, as its node based workflow and data flow system required a completely different approach compared to other software.

Another challenge was achieving believable destruction behaviour. Early simulations produced unrealistic debris sizes and movement, which required rebuilding parts of the node network and refining fracture settings and constraints.

Integrating the Houdini Digital Asset for engine use also presented issues, as some parameters caused instability or failed to transfer correctly. This required simplifying the system and focusing on reliable controls.

Material and texture issues also occurred, including flickering and rendering problems, which led to removing textures in order to maintain stability and complete the core VFX objectives.

Outcome

The Tower project demonstrates my understanding of procedural VFX workflows and real time destruction systems using Houdini.

It highlights my ability to combine multiple simulation techniques into a single cohesive system while maintaining performance awareness and technical stability. The project also reflects my ability to learn complex software quickly and apply industry relevant workflows such as procedural asset creation and engine integration.

This project strengthened my skills in Houdini, procedural problem solving, simulation design and technical art pipelines, providing a strong foundation for future work in VFX and game development.
