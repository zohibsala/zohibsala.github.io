Blaze Ball

Overview

Blaze Ball is a local multiplayer football game developed in Unity using C#. The project was created as part of a group coursework and is designed for up to four players competing in short and fast paced matches.

The game focuses on a simple but engaging gameplay loop where two teams compete to score goals within a limited time. The environment is a small enclosed pitch designed to keep the action constant and maintain a high tempo throughout each match.

My Role

I contributed to both the gameplay systems and user interface design. My responsibilities included developing core scripts for player interaction, implementing menu systems and designing the overall visual layout of the game's interface.

Key Features

  • Local multiplayer for up to four players using keyboard input
  • Physics based ball movement for realistic gameplay interaction
  • Scoring system to track goals and determine the winner
  • Game timer system controlling match duration
  • Player interaction system for picking up and controlling the ball
  • Main menu and pause system for smooth navigation
  • Game over screen displaying match results
  • Sound effects for actions such as scoring goals
  • Environment design featuring a small pitch surrounded by natural obstacles

Technical Implementation

The project was developed in Unity using C#. I implemented several core gameplay systems including the player interaction system, which allows players to pick up, control and interact with the ball during matches.

The ball uses physics based movement to create more dynamic and unpredictable gameplay. This required careful tuning to ensure the ball felt responsive while remaining balanced for competitive play.

I also developed key UI systems including the main menu, pause menu, game timer and game over screen. These systems were designed to ensure a smooth user experience and clear feedback during gameplay.

Multiplayer input was handled through a shared keyboard setup, allowing multiple players to control different characters within the same session.

Challenges

One of the main challenges was managing multiplayer input on a single keyboard, ensuring that controls were responsive and did not conflict between players.

Another challenge was balancing the physics of the ball to make gameplay feel smooth and enjoyable while maintaining fairness for all players.

Working in a team environment also required clear communication and coordination, especially when integrating different systems and ensuring compatibility between scripts.

Outcome

Blaze Ball demonstrates my ability to work collaboratively on a multiplayer game project while contributing to both gameplay mechanics and user interface design.

The project strengthened my skills in Unity development, C# scripting, UI design and teamwork. It also highlights my ability to build interactive systems that support fast paced and engaging gameplay experiences.
