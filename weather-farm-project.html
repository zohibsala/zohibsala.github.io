Weather Farm

Overview

Weather Farm is a mobile friendly farming management game developed in Unity as a playable vertical slice for Android and Windows platforms. The project was based on a brief provided by Frontier Developments and focuses on integrating real world data into gameplay.

The game is inspired by titles such as Hay Day and Clash of Clans by Supercell, but introduces a key difference where real world weather and date data directly affect how crops grow and how the player manages their farm.

The core gameplay loop is designed for short sessions where players plant crops, wait for them to grow and harvest them to earn coins, creating a simple but engaging management experience.

My Role

I worked as one of the Scrum Masters and contributed as a programmer. I was responsible for implementing core systems such as weather logic, crop behaviour and the shop system, while also supporting team coordination and task management throughout development.

Key Features

  • Real world weather system using live data from Yorkshire in the United Kingdom
  • Dynamic crop growth affected by weather conditions such as sunny, rainy, snowy and windy
  • Simple gameplay loop focused on planting, waiting and harvesting
  • Economy system with coins used to purchase seeds and expand the farm
  • JSON based save and load system to store player progress locally
  • Shop system for selecting and buying crops
  • Weather visualisation using Unity particle effects
  • Mobile friendly design with short 5 to 15 minute play sessions

Technical Implementation

The project was developed in Unity using C#. One of the main technical features was integrating real world weather and date data into the game logic, allowing environmental conditions to directly influence crop behaviour.

I implemented systems that adjust crop growth based on different weather states. For example, rain allows crops to grow automatically while other conditions can slow down progression. This required conditional logic to ensure each weather type affected gameplay in a clear and consistent way.

A JSON based save and load system was developed to store player progress locally. This allowed the game state, including crops and currency, to persist between sessions without relying on external services.

Weather effects were visualised using Unity particle systems to represent rain, snow and wind, helping to reinforce the connection between gameplay systems and the environment.

Challenges

One of the main challenges was integrating real world weather data and ensuring it correctly influenced gameplay systems. This required testing different conditions and making sure the logic remained stable and consistent.

Another challenge was managing technical issues within Unity, including bugs related to crop positioning, UI interaction and system behaviour. These were resolved through debugging, testing and refining scripts.

Time constraints also required prioritising core features over additional content, ensuring the game remained functional and polished within the given deadline.

Outcome

Weather Farm demonstrates my ability to develop a data driven game system that connects real world information with gameplay mechanics. It also highlights my experience working in a team using Agile and Scrum methods while contributing to both technical and creative aspects of development.

The project strengthened my skills in Unity, C# programming, system design, problem solving and teamwork, and provided valuable experience in building a complete and playable prototype from concept to final delivery.
