Alex Blue

Overview

Alex Blue is a real time character animation project focused on motion capture, animation blending and player control in Unity. The project centres around a fully playable third person character designed for an action adventure style environment.

The goal was to create a responsive and believable character using motion captured animation, while ensuring smooth transitions between movement states and direct interaction with player input.

My Role

I was responsible for the full character pipeline including modelling, rigging, motion capture integration, animation setup and gameplay controls.

Key Features

  • Playable third person character with real time movement and control
  • Motion captured animations including idle, walk, run, jump and emote
  • Animation system built using Unity Animator and blend trees
  • Smooth transitions between movement states based on player input
  • Keyboard control system using WASD, Shift, Spacebar and input triggers
  • Low poly character design focused on silhouette and readability
  • Mixamo auto rigging for fast and reliable humanoid setup

Technical Implementation

The character was modelled in Blender using a simple and clean low poly style to ensure good performance and clear visual readability. The design avoids detailed facial features and instead relies on movement and silhouette to express personality.

Rigging was completed using Mixamo auto rigging, which generated a stable humanoid skeleton compatible with Unity. This allowed for efficient retargeting of animation data and avoided the need for complex manual weight painting.

Motion capture data was recorded using Axis Studio and then cleaned and prepared before being imported into Unity. The animations were tested and adjusted to ensure they worked correctly with the rig and maintained a natural sense of movement.

Inside Unity, I built the animation system using an Animator Controller with defined states for idle, walk, run, jump and emote. A blend tree was used to smoothly transition between movement states based on player input, creating responsive and fluid character control.

The player control system was implemented using a simple keyboard layout, allowing movement with WASD, running with Shift, jumping with Spacebar and triggering emotes through input actions. The animation system was directly linked to these inputs to ensure consistent behaviour.

Challenges

One of the main challenges was dealing with inconsistencies in motion capture data, particularly jittering and instability in certain animations such as running. This required multiple recordings and adjustments to improve the final result.

Another challenge was ensuring smooth animation blending inside Unity. This involved fine tuning transition timings and parameters to avoid unnatural or abrupt changes between states.

There were also technical issues related to control responsiveness and animation triggering, which required debugging and refinement of both the movement script and Animator setup.

Outcome

Alex Blue demonstrates my understanding of a real time character pipeline from modelling and rigging to motion capture integration and gameplay control.

The project strengthened my skills in Blender, Unity and animation systems, particularly in working with motion capture data and building responsive character controllers. It also reflects my ability to solve technical problems and create a functional and playable character system suitable for any environment.
